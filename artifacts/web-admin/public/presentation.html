Smart Auto-Select Logic

If the volunteer's currently selected hostel is no longer in their assigned list (stale), the app automatically snaps to their new primary hostel — no manual action needed.

Offline Fallback

Last known user profile and all GET responses are cached in AsyncStorage. If the network drops, the app stays functional using the cached data.

Security

Security Architecture

🔑
JWT Authentication
Every request must carry a signed JWT token. Tokens are verified server-side on every route via requireAuth middleware. No session storage — stateless.
🛡️
Role-Based Access Control (RBAC)
requireStudent / requireVolunteer / requireAdmin / requireSuperAdmin middleware gates every API route. Role is read from the DB on each request, not trusted from the token payload alone.
🏠
Hostel-Scoped Data
Volunteers can only read/write students in their assigned hostel. Coordinators are scoped to their assigned hostel list. Cross-hostel data access is blocked at the query level.
🚦
Rate Limiting
General: 2,000 requests / 15 min per IP. Auth endpoints: 50 requests / 15 min. Prevents brute-force login attacks and API abuse.
🔒
Inventory Lock (Write-Once)
Once submitted, inventory records are permanently locked. Server returns HTTP 403 on any PATCH attempt — no client-side workaround possible.
📸
Screenshot Detection Logging
Android screenshot listener fires on any screen capture. Event is immediately logged to the activity log with the user's name, email, and timestamp — visible to superadmin in real time.
📤
File Upload Limits
Multer enforces 5 MB max on CSV imports. Only multipart/form-data accepted for file routes. Server validates CSV format and rejects malformed rows with detailed error messages.
🌐
mTLS Proxy (Replit)
All traffic to the dev server is proxied through Replit's mTLS gateway. Production runs on Railway with HTTPS enforced. Trust proxy enabled for accurate IP rate limiting.

⚡ gzip Compression

All API responses compressed with the compression middleware. Dramatically reduces payload size for large student lists and reports over mobile networks.

🔄 Cluster Mode

Express server runs in Node.js cluster mode. Worker processes share the port, enabling multi-core CPU utilisation on the server without any code changes.

All list endpoints are paginated with limit + offset. Infinite scroll on mobile — next page loads automatically as the user scrolls near the end of the list.
